Executive Summary

First off, I try to assess the feasibility of unifying these 5 Business Units (Mtech, MKitchen, Marathon Express, MTrading, ShweZay) with fragmented data systems. This initiative will create a unified data warehouse to enable Cross-BU analytics, standardized master data, and executive-level business intelligence.

🎯 Project Objectives

  • Standardize Unit of Measure (UOM) conversions

⚠️ Current State Challenges

Click on each problem card to learn more about the specific challenges we face.

SKU Mismatch

Same physical products have different SKUs across Business Units, making group-wide analysis impossible.

Weak UOM Mapping Standard

Different units of measure (pieces, cartons, pallets) need proper conversion mappings based on product keys.

Separate AP/AR

Financial records kept separately by BU prevent consolidated cash flow visibility and supplier negotiation leverage.

Limited Batch Traceability

Batch and expiry tracking exists but is isolated per BU, preventing cross-BU optimization of expiring stock.

Data Silos

Each BU operates independently with no group-wide visibility into inventory, sales, or customer relationships.

Finance-Focused System

DAS software emphasizes financial transactions but lacks operational workflows for ground-level adjustments.

✅ Proposed Solution

Unified SKU System

Create a master product catalog with group-wide SKUs and mapping tables to translate BU-specific codes.

Standard UOM Conversion

Implement a conversion matrix that standardizes all measurements to base units for accurate comparisons.

Consolidated AP/AR

Unify financial records to show total exposure per supplier/customer across all BUs for better negotiation.

Enhanced Batch Tracking

Cross-BU batch visibility with FEFO (First Expired, First Out) logic and automated expiry alerts.

Cross-BU Analytics

Group-wide dashboards showing consolidated inventory, sales, and customer insights across all BUs.

Automated ETL Pipeline

Python-based data extraction, transformation, and loading with daily synchronization from DAS to analytics DB.

🗄️ Database Schema Design

Star schema design optimized for analytical queries and BI tools.

Dimension Tables

dim_product (Product Master)

Column Type Description
PK product_sk SERIAL Surrogate key
clean_sku VARCHAR(50) Unified product code
product_name VARCHAR(255) Product description
base_uom VARCHAR(20) Standard unit (PCS, KG, L)
category_id INT Product category

dim_location (BU & Branches)

Column Type Description
PK location_sk SERIAL Surrogate key
bu_name VARCHAR(50) Business Unit name
branch_code VARCHAR(50) Shop/warehouse code
branch_name VARCHAR(100) Branch description
location_type VARCHAR(20) Warehouse/Retail/Transit

dim_batch (Batch & Expiry Tracking)

Column Type Description
PK batch_sk SERIAL Surrogate key
batch_number VARCHAR(50) Batch identifier
manufacturing_date DATE Production date
expiry_date DATE Expiration date
days_to_expiry INT Calculated field

Fact Tables

fact_sales (Sales Transactions)

Column Type Description
PK sale_id SERIAL Transaction ID
FK date_sk DATE Transaction date
FK location_sk INT Which BU/Shop
FK product_sk INT Which product
FK customer_sk INT Which customer
FK batch_sk INT Which batch
quantity_base_uom DECIMAL(15,4) Standardized quantity
unit_price DECIMAL(15,4) Price per unit
total_revenue DECIMAL(15,4) Total sale amount
total_cost DECIMAL(15,4) Cost of goods sold
gross_profit DECIMAL(15,4) Revenue - Cost

fact_inventory_batch (Inventory by Batch)

Column Type Description
date DATE Snapshot date
FK location_sk INT Warehouse/Shop
FK product_sk INT Product
FK batch_sk INT Batch number
quantity_on_hand DECIMAL(15,4) Current stock
days_to_expiry INT Days until expiry

Mapping Tables

map_product_sku (SKU Translation)

Column Type Description
das_native_sku VARCHAR(100) Original SKU from DAS
das_bu_id INT Which BU uses this SKU
FK clean_product_sk INT Mapped to unified product
conversion_factor DECIMAL(10,4) UOM conversion multiplier
